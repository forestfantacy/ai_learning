# 质量闸门

在相信计算结果、归因结论或行动建议前，使用这些检查。

## 闸门一：指标契约

计算任何关键指标前，先明确：

| 契约项 | 要澄清什么 |
|-|-|
| 业务对象 | 订单、用户、商品、渠道、供应商、账号等。 |
| 数据粒度 | 日、周、月、订单级、用户级、SKU 级等。 |
| 分子 | 统计或求和的对象是什么。 |
| 分母 | 如果是比率，分母是什么。 |
| 时间窗口 | 明确分析时间范围。 |
| 时间口径 | 下单时间、支付时间、退款时间、事件时间等。 |
| 过滤条件 | 排除哪些状态、测试记录、取消订单、退款、重复行、机器人流量等。 |
| 权威数据源 | 哪张表或哪个系统是准的。 |
| 口径风险 | 历史定义变更、字段缺失或同步问题。 |

## 闸门二：统计严谨性

检查：

- 样本量：小样本必须降低置信度。
- 对比公平性：时间、分组和业务条件是否可比。
- 多重比较：切分维度越多，越容易碰巧发现异常。
- 效应大小：统计上看得见，不等于业务上值得行动。
- 不确定性：优先给出范围或置信表达，不要给无依据的点估计确定性。

## 闸门三：可追溯性

在最终报告中分开说明：

| 层级 | 包含内容 |
|-|-|
| 数据提取 | 使用了哪些表、字段、筛选条件、源文件、SQL 或脚本。 |
| 数据处理 | 清洗、关联、排除、派生指标和聚合粒度。 |
| 业务解释 | 哪些是事实、哪些是推断、哪些是假设、还有哪些替代解释。 |

## 闸门四：结论压力测试

推荐行动前，检查：

1. 换成周维度或月维度，结论是否仍成立？
2. 去掉极端记录后，结论是否仍成立？
3. 在关键分层下，结论是否仍成立？
4. 换一个合理分母后，结论是否仍成立？
5. 与去年同期、上个周期或可比对照组相比，结论是否仍成立？

如果结论只在一种脆弱切法下出现，必须标注为探索性发现。

## 闸门五：图表适配

图表必须回答一个具体问题。

| 问题 | 推荐图表 |
|-|-|
| 看时间趋势 | 折线图 |
| 比较不同类别 | 条形图 |
| 看分布或离群点 | 直方图、箱线图 |
| 看两个指标关系 | 散点图 |
| 看构成变化 | 堆叠条形图、占比图 |
| 看漏斗流失 | 漏斗图 |
| 看留存 | Cohort 热力图 |

避免：

- 只展示百分比，不展示分母。
- 把日、周、月不同粒度混在一张图里。
- 为了装饰而放图，图表没有支持任何决策。

## 结论标签

使用明确标签：

- **事实**：直接由数据计算得到。
- **推断**：有证据支持的可能解释。
- **假设**：合理但尚未验证。
- **风险**：可能推翻结论的因素。
- **缺失数据**：验证所需但当前没有的字段或数据源。
