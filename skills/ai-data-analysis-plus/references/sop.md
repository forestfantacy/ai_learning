# SOP 参考

## 一句话原则

Excel 时代的分析，前提是分析师已经知道该问什么问题。AI 时代的分析，应该帮助用户发现问题、搭建路径、验证证据，并把结论转成行动。

## 阶段一：澄清决策目标

真正的决策目标要回答四个问题：

| 问题 | 含义 |
|-|-|
| 谁要使用这份分析？ | 老板、运营、销售、产品、财务、供应链等。 |
| 他要做什么判断？ | 诊断下降、分配预算、确定商品优先级、管理风险等。 |
| 分析会影响什么动作？ | 预算、活动、商品结构、库存、成本控制、预警、人力安排等。 |
| 不同结论是否会触发不同动作？ | 如果不会，目标通常还不够清晰。 |

弱目标：“分析销售情况。”

强目标：“判断 3 月销售额为什么下降，并决定优先介入哪个渠道或类目。”

## 阶段二：目标发现

当用户只给了一堆表，或只说“分析一下”，不要立刻做全量分析。先生成候选分析目标。

常见候选目标：

| 候选目标 | 业务问题 | 典型使用者 |
|-|-|-|
| 销售增长诊断 | 销售额是否下降或增长，变化来自哪里？ | 老板、运营 |
| 渠道效率分析 | 哪些获客渠道高价值、低退货、高毛利、值得加预算？ | 增长、投放 |
| 商品/类目结构分析 | 哪些类目拉动销售、拖累毛利、依赖折扣或带来退货？ | 商品运营 |
| 客户价值分析 | 不同客户分层在客单价、留存、复购、退货风险上有何差异？ | 用户运营 |
| 供应商或供给质量分析 | 供应商评级是否影响毛利、退货、库存或商品表现？ | 采购、供应链 |

候选目标排序标准：

1. 业务价值：是否影响收入、利润、成本、风险或战略重点。
2. 数据支持度：当前字段是否足以回答。
3. 可行动性：分析后是否有人能采取动作。
4. 异常信号：数据里是否已经出现波动或离群点。
5. 决策对象匹配度：结果是否对应目标使用者关心的问题。

## 阶段三：还原业务模型

解释数字前，先还原业务模型。

1. 识别粒度：
   - 订单表：一行是一笔订单，还是一个订单明细行？
   - 客户表：一行是否代表一个客户？
   - 商品表：一行是否代表一个 SKU？
   - 供应商表：一行是否代表一个供应商？
2. 分类字段：

| 层级 | 示例 | 作用 |
|-|-|-|
| 结果指标 | 销售额、实付金额、毛利、毛利率 | 判断业务结果好坏 |
| 过程指标 | 订单量、客单价、购买件数、折扣、退货率 | 解释结果变化 |
| 基础字段 | 客户 ID、商品 ID、数量、成本价、单价 | 构造指标 |
| 分析维度 | 月份、渠道、城市、客户等级、类目、供应商评级 | 定位变化来自哪里 |

3. 建立指标关系：
   - 销售额 = 订单量 * 客单价。
   - 客单价 = 实付金额 / 订单量。
   - 毛利 = 实付金额 - 数量 * 成本价。
   - 毛利率 = 毛利 / 实付金额。
   - 退货率 = 退货订单数 / 总订单数。

## 阶段四：异常检测

肉眼看趋势图只能作为起点。

| 检查 | 方法 | 要回答的问题 |
|-|-|-|
| 趋势异常 | 环比、移动平均 | 是否偏离近期趋势？ |
| 分位数异常 | 历史 P10/P90、IQR | 是否落入历史极端区间？ |
| 统计异常 | Z-score、控制线 | 是否超过正常波动范围？ |
| 业务规则异常 | 指标逻辑校验 | 指标组合是否不符合业务常识？ |

业务规则异常示例：

- 订单量上升但销售额下降：可能是客单价、折扣或商品结构恶化。
- 销售额上升但毛利下降：可能是低毛利商品占比提高。
- 转化率上升但订单量下降：可能是流量入口减少。
- 退货率突然升高：可能涉及商品质量、物流、活动人群或退货政策变化。

样本期很短时，必须降低趋势判断置信度，并明确说明。

## 阶段五：归因证据链

归因不是“哪个维度下降最大，哪个维度就是原因”。要把归因当成四步证据链。

1. **定位影响**
   - 哪个结果指标变了？
   - 变化幅度是多少？
   - 变化来自数量、价格/客单价、结构、毛利、退货，还是其他过程指标？

2. **生成候选原因**
   - 内部可控因素：预算、页面转化、库存、商品结构、价格、折扣、运营动作、供应商质量。
   - 数据或流程因素：指标定义、退款、重复行、时区/时间口径变化、缺失数据。
   - 外部因素：市场需求、竞品活动、平台流量、季节性、政策、宏观环境。

3. **用对照验证**
   - 时间一致性：候选原因是否发生在变化之前或变化期间？
   - 对象特异性：只影响部分渠道/类目/客户，还是全体一起变化？
   - 对照组：有没有相似对象没有下降或没有变化？
   - 内部数据解释力：在引入场外因素前，内部指标是否已经能解释变化？

4. **输出置信度和下一步证据**
   - 高置信度：当前数据同时支持影响位置、作用机制和对照验证。
   - 中置信度：影响和位置清楚，但机制还需要更多数据。
   - 低置信度：信号弱、样本小，或替代解释同样成立。

## 阶段六：预测与行动闭环

每个有价值的结论都要完成三层转化：

| 层级 | 示例 |
|-|-|
| 数据事实 | 3 月销售额相比 1 月下降 X。 |
| 业务判断 | 下降主要来自订单量减少，集中在搜索和线下渠道。 |
| 业务动作 | 在调整预算前，先核查渠道漏斗和类目供给数据。 |

办公场景常用预测方法：

| 方法 | 适用场景 |
|-|-|
| 趋势外推 | 指标有足够历史且趋势相对稳定。 |
| 因子拆解预测 | 结果指标有清晰公式，例如流量 * 转化率 * 客单价。 |
| 情景分析 | 决策不确定性高，需要乐观、基准、悲观三种情况。 |

行动建议必须包含：

| 要素 | 要求 |
|-|-|
| 目标问题 | 明确指标和缺口。 |
| 建议动作 | 具体下一步，而不是“加强”“优化”。 |
| 负责人 | 角色或团队。 |
| 预期效果 | 有依据时给出方向和大致幅度。 |
| 验证指标 | 用什么指标判断动作有效。 |
| 风险限制 | 哪些因素可能让建议失效或受限。 |
